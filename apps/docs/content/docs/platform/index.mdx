Capsule Pro is a monorepo-based platform built with Next.js 15, React 19, and TypeScript. It uses pnpm workspaces with Turborepo for build orchestration.

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Framework | Next.js 15 (App Router) |
| Language | TypeScript 5.9 |
| Database | Neon Serverless Postgres via Prisma 7 |
| Authentication | Clerk |
| Payments | Stripe |
| Realtime | Ably (transactional outbox pattern) |
| Collaboration | Liveblocks |
| AI | OpenAI via Vercel AI SDK |
| UI | shadcn/ui + Radix UI + Tailwind CSS v4 |
| Email | Resend + React Email |
| Monitoring | Sentry + BetterStack/Logtail |
| Analytics | PostHog + Vercel Analytics |
| Deployment | Vercel |

## Core Patterns

- **IR-First Architecture** — The Manifest DSL compiles to an Intermediate Representation that governs all domain mutations
- **Multi-Tenancy** — Every tenant-scoped model uses composite primary keys `[tenantId, id]` with automatic scoping middleware
- **Transactional Outbox** — Domain events are written atomically with mutations, then published to Ably by a separate process
- **Vendor Abstraction** — Every external service is wrapped in a `@repo/*` package for swappability
- **Soft Deletes** — All models use `deletedAt` timestamps instead of hard deletes
- **Status State Machines** — Formal state transitions stored in database tables with role-based guards
