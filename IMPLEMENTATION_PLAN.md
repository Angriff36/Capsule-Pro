# IMPLEMENTATION_PLAN.md — 3-Command Manifest Trio

> **North star:** a 3-command setup — `pnpm manifest:validate` → `pnpm manifest:compile` → `pnpm manifest:build` — that fully regenerates the exact manifest-derived implementation the running app consumes, byte-identically, with no hand-editing of `*- DO NOT EDIT` artifacts and no ad-hoc side commands. Today `build.mjs` produces only 6 of the 12 consumed artifact classes; the other 6 require manual `manifest:client` / `manifest:ir:embed` / `manifest:generate-metadata` / `manifest:openapi` / `manifest:registries` / `manifest:command-schemas` invocations that are easy to skip and drift.
>
> **Verification status legend:** ✅ verified-disposition (2026-06-26, file:line evidence in log) · ➕ new finding · ⏳ genuine fork (owner sign-off).

**Scope boundary.** IN SCOPE: consolidate Capsule's own `manifest:*` wrapper scripts into the trio; fold consumed-artifact producers into `manifest:build`; source-level validation; validator unification; dead-script retirement; doc completion. OUT OF SCOPE (separate branch-isolated migration `manifest/full-official-generation`, governed by `manifest/IMPLEMENTATION_PROMPT.md` + `manifest/task_plan_full_official_generation.md` + `manifest/phase-out-registry.md` + `manifest/notes.md`): driving the full Prisma schema/stores from IR via official projections, retiring hand-authored `schema.prisma`. Reference it; do NOT pull it in. Do not conflate the two.

> **Note on package layout (verified):** all `manifest:*` scripts live in the **root `package.json`**. There is no `manifest/package.json` — `manifest/runtime/package.json` is the `@repo/manifest-runtime` workspace package. Edits to `manifest:*` script entries go in root `package.json`.
