# Manifest — Capsule Pro Convex Clone

> **Architecture:** compile-to-Convex. Registry: `manifest/convex-phase-out-registry.json`

Read `constitution.md` first. This file is operational detail for agents working in `manifest/`.

---

## The model (not runtime interpreter)

```txt
.manifest  →  compile  →  IR  →  ConvexProjection  →  convex/
```

Governance is **baked into generated mutations**, not executed by `RuntimeEngine` at request time.

Each generated mutation runs:

```txt
policies → guards → constraints → mutate → event row → inline reactions
```

Reactions are **inline fan-out in the same transaction** (not schedulers). Schedules go to `convex/crons.ts`. Webhooks to `convex/http.ts`. Sagas to `convex/sagas.ts`.

## Commands

| Script | Purpose |
|--------|---------|
| `pnpm manifest:compile` | `.manifest` → `manifest/ir/kitchen.ir.json` |
| `pnpm manifest:generate-convex` | IR → all six `convex/` surfaces |
| `pnpm manifest:build` | compile + generate-convex + registries |
| `pnpm manifest:check-convex-drift` | CI: committed convex/ matches IR |
| `pnpm manifest:enforce-convex-architecture` | CI: no new Prisma/runtime-interpreter patterns |
| `pnpm manifest:ci` | Full governance chain |

## Forbidden in new work

- `storeProvider`, `createManifestRuntime`, `prismaOverride`
- `@repo/database`, `@prisma/client`
- Hand-written `convex/*.ts` without `GENERATED by the Manifest → Convex projection`
- Next.js domain API routes (`apps/api/app/api/**` except temporary shims)
- `/integration/prisma` projection workflow

## Frontend integration (target)

```ts
// Write — call generated mutation (name pattern: Entity_command)
await convex.mutation(api.mutations.Event_create, { title, guestCount, ... });

// Read — reactive query
const events = useQuery(api.queries.Event_list, { tenantId });
```

Do **not** add new `POST /api/manifest/...` or `apiFetch` callsites.
