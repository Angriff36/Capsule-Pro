# `manifest/runtime/` — Runtime dispatcher sidecars

The `@repo/manifest-runtime` workspace package plus the generated runtime
sidecars. All sidecars live **here** now (the legacy `packages/manifest-ir/`
location was retired 2026-06-03):

| File | Purpose | Read by |
|---|---|---|
| `commands.registry.json` | Flat command index (952 entries) | `src/command-resolver.ts` via `src/commands-registry.ts` (`import "../commands.registry.json"`) → re-exported to the dispatcher at `apps/api/lib/manifest/command-resolver.ts` |
| `routes.manifest.json` | Generated route manifest | `packages/mcp-server/src/lib/route-conformance-scan.ts` |
| `routes.ts` | Generated route helpers | route helper re-export chain |

These are generated by the codegen scripts in `manifest/scripts/` from the IR in
`manifest/ir/`. Per constitution §4a this is the canonical home for runtime
artifacts; `packages/manifest-runtime/`, `packages/manifest-ir/`, and
`packages/manifest-adapters/` are forbidden resurrection paths.

## Command surface

`commands.registry.json` now carries the **full** command surface (952 entries),
not a kitchen-only subset. The broader governance registry lives at
`manifest/governance/commands.json`. `enforce-surface` flags drift between them
and is currently in REPORTING mode (see `.github/workflows/manifest-ci.yml`).
